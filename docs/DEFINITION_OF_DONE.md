# Definition of Done (DoD)

---

## DoD-CODE: Kód Deliverable

Egy kód deliverable DONE, ha:

### Funkcionális
- [ ] Megvalósítja a specifikált funkcionalitást
- [ ] Minden Acceptance Criteria teljesül
- [ ] Edge case-ek kezelve

### Minőségi
- [ ] Self code review megtörtént
- [ ] Nincs TODO/FIXME/HACK a kódban
- [ ] Coding standards betartva
- [ ] Megfelelő error handling van

### Tesztelés
- [ ] Unit tesztek írva (ha alkalmazható)
- [ ] Tesztek átmennek
- [ ] Teszt coverage megfelelő

### Dokumentáció
- [ ] Kód kommentek naprakészek
- [ ] README frissítve (ha szükséges)
- [ ] Használati példák vannak

### Integráció
- [ ] Nincs merge conflict
- [ ] Build sikeres
- [ ] Nincs regresszió

### Validáció
- [ ] Validator Agent jóváhagyta
- [ ] Minden CRITICAL/MAJOR hiba javítva

---

## DoD-DOC: Dokumentáció Deliverable

Egy dokumentum DONE, ha:

### Tartalom
- [ ] Célját egyértelműen leírja
- [ ] Teljes (nem hiányos szekciók)
- [ ] Pontos és naprakész információk

### Formátum
- [ ] Markdown formátum helyes
- [ ] Konzisztens struktúra
- [ ] Táblázatok, listák helyesen formázva

### Érthetőség
- [ ] Célközönség számára érthető
- [ ] Nincs zsargon magyarázat nélkül
- [ ] Példák ahol szükséges

### Review
- [ ] Self review megtörtént
- [ ] Helyesírás ellenőrizve
- [ ] Linkek működnek

### Validáció
- [ ] Validator Agent jóváhagyta

---

## DoD-SCRIPT: Script Deliverable

Egy script DONE, ha:

### Funkcionális
- [ ] Végrehajtható
- [ ] Specifikált funkciót ellátja
- [ ] Hibakezelés megfelelő

### Dokumentáció
- [ ] Header comment (cél, használat, paraméterek)
- [ ] Verzió szám szerepel
- [ ] Példa használat dokumentálva

### Robusztusság
- [ ] Ellenőrzi az előfeltételeket
- [ ] Értelmes hibaüzenetek
- [ ] Nem hagy szennyezett állapotot hiba esetén

### Validáció
- [ ] Tesztelve különböző bemenetekkel
- [ ] Validator Agent jóváhagyta
